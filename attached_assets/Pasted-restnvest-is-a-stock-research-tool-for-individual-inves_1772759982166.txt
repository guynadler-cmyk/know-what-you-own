TOKEN-GATE LOGIC
Before rendering the page, read the ?token= URL parameter and check it against a Firestore collection called inviteTokens. Each document in that collection has a field active: true/false and uses the token string as the document ID.

If no token is present → redirect to https://restnvest.com/app
If token exists but active: false → redirect to https://restnvest.com/app
If token is valid and active: true → render the founding page normally

The working invite URL will be: restnvest.com/founding?token=founding2026

TECHNICAL REQUIREMENTS

Firebase Authentication: both Email/Password and Google sign-in
Firebase Firestore for inviteTokens collection and user profile storage
Use Firebase SDK v9+ (modular)
Protect the /founding route — invalid or missing token redirects to https://restnvest.com/app
If user is already authenticated and hits /founding, redirect straight to https://restnvest.com/app
Store Firebase config in environment variables (.env): VITE_FIREBASE_API_KEY, VITE_FIREBASE_AUTH_DOMAIN, VITE_FIREBASE_PROJECT_ID, VITE_FIREBASE_APP_ID
Use React + Vite or plain HTML/JS — whichever is already in the project
Mobile responsive
